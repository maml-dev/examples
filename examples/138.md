# 138 - Monitoring Config

Application monitoring and alerting setup. This is an example of a [MAML](https://maml.dev) document.

```maml
{
  service: "functionalities"
  # Prometheus metrics
  metrics: {
    endpoint: "/metrics"
    interval: 30 # seconds
    labels: {
      env: "staging"
      region: "eu-west"
      team: "baby"
    }
  }

  # Alert thresholds
  alerts: {
    cpu_threshold: 89
    memory_threshold: 84
    latency_p99_ms: 308
    error_rate: 0.02
    notification: {
      channel: "#incidents"
      severity: "info"
    }
  }
}
```
