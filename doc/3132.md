# 3132 - Monitoring Config

Application monitoring and alerting setup. This is an example of a [MAML](https://maml.dev) document.

```maml
{
  service: "infrastructures"
  # Prometheus metrics
  metrics: {
    endpoint: "/metrics"
    interval: 60 # seconds
    labels: {
      env: "staging"
      region: "us-east"
      team: "electronics"
    }
  }

  # Alert thresholds
  alerts: {
    cpu_threshold: 90
    memory_threshold: 85
    latency_p99_ms: 110
    error_rate: 0.01
    notification: {
      channel: "#incidents"
      severity: "info"
    }
  }
}
```
