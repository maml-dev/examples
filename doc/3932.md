# 3932 - Monitoring Config

Application monitoring and alerting setup. This is an example of a [MAML](https://maml.dev) document.

```maml
{
  service: "architectures"
  # Prometheus metrics
  metrics: {
    endpoint: "/metrics"
    interval: 60 # seconds
    labels: {
      env: "staging"
      region: "ap-south"
      team: "health"
    }
  }

  # Alert thresholds
  alerts: {
    cpu_threshold: 73
    memory_threshold: 75
    latency_p99_ms: 451
    error_rate: 0.05
    notification: {
      channel: "#ops-alerts"
      severity: "critical"
    }
  }
}
```
